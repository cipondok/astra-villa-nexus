# ASTRA Villa — Founder Daily Execution Routine

> Disciplined daily rhythm to maximize product progress, market traction, and business growth.

---

## Philosophy

Every day must move **three needles simultaneously**: product quality, marketplace supply, and revenue pipeline. The founder's calendar is the company's strategy made visible.

**Rule of 3**: Each day, identify exactly 3 outcomes that would make the day a success — one product, one market, one strategic.

---

## Daily Execution Structure

### 🌅 Morning — Strategic Control Block (06:00–09:00)

**Purpose**: Gain situational awareness and set the day's direction.

| Time | Activity | Tool/Resource |
|------|----------|---------------|
| 06:00–06:30 | Review overnight metrics | Admin Command Center dashboard |
| 06:30–07:00 | Check AI system health & deal alerts | `usePlatformHealth`, `useAISystemHealth` hooks |
| 07:00–07:30 | Triage support tickets & critical alerts | Admin Alerts panel |
| 07:30–08:00 | Define today's **#1 product task** | `.lovable/plan.md` or task board |
| 08:00–08:30 | Plan one high-impact growth action | Partnership tracker / CRM |
| 08:30–09:00 | Strategic reading or competitor scan | Industry newsletters, ProductHunt |

**Morning Checklist** (non-negotiable):
- [ ] Platform uptime confirmed (all subsystems operational)
- [ ] New listing count reviewed (target: 50+/day by Month 6)
- [ ] Investor signup trend checked (target: 20+/day by Month 6)
- [ ] Deal alerts accuracy spot-checked (1–2 random alerts)
- [ ] Daily 3 outcomes written down

**Key Metrics to Review**:
```
├── New Properties (24h)     → Growth signal
├── Active Users (24h)       → Engagement signal
├── AI Jobs Completed (24h)  → System health signal
├── Deal Alerts Generated    → Intelligence quality signal
├── Support Tickets (open)   → Customer health signal
└── Revenue Events (24h)     → Business health signal
```

### 🏗️ Midday — Product Building Block (09:00–14:00)

**Purpose**: Deep work on platform evolution. This is the most valuable block.

| Time | Activity | Focus Area |
|------|----------|------------|
| 09:00–09:30 | Sprint planning / PR review | GitHub + Lovable |
| 09:30–12:00 | **Deep building session** | Feature development, AI modules, UI |
| 12:00–12:30 | Lunch break | — |
| 12:30–13:30 | Testing & QA validation | Preview environment, edge cases |
| 13:30–14:00 | Deploy & monitor | Production deployment |

**Building Priorities Framework** (choose daily focus):

| Priority Level | Category | Examples |
|----------------|----------|----------|
| P0 — Critical | Revenue blockers | Payment flow bugs, subscription gate failures |
| P1 — High | Intelligence depth | AI scoring accuracy, deal detection improvements |
| P2 — Medium | User experience | Dashboard modernization, mobile optimization |
| P3 — Low | Future features | New marketplace modules, advanced analytics |

**Daily Product Rules**:
1. Ship at least **one visible improvement** every day
2. Never spend >2 hours on a bug without escalating approach
3. Test on mobile viewport before marking complete
4. Update `.lovable/plan.md` with progress notes
5. Protect this block from meetings — it's sacred

**Technical Focus Rotation** (weekly cycle):
- **Monday**: AI Intelligence modules (scoring, predictions, deal engine)
- **Tuesday**: Marketplace features (listings, search, filters)
- **Wednesday**: Admin & analytics dashboards
- **Thursday**: Mobile experience & performance optimization
- **Friday**: Infrastructure, security, technical debt

## Weekly High-Impact Rituals

### Monday — Week Launch (30 min)
- [ ] Review previous week's 3 key metrics (listings, users, revenue)
- [ ] Set 3 weekly outcomes (product, market, strategic)
- [ ] Prioritize feature backlog for the week
- [ ] Schedule critical meetings/calls

### Wednesday — Mid-Week Pulse (20 min)
- [ ] Check progress against weekly outcomes
- [ ] Adjust priorities if blocked
- [ ] Review AI system accuracy metrics
- [ ] Spot-check marketplace quality (random listing audit)

### Friday — Week Close & Reflection (45 min)
- [ ] Score weekly outcomes (achieved / partial / missed)
- [ ] Document key learnings and decisions
- [ ] Update growth metrics dashboard
- [ ] Plan weekend content and Monday priorities
- [ ] Celebrate one win (no matter how small)

### Monthly — Strategic Review (2 hours)
- [ ] Full platform health assessment
- [ ] Revenue vs. target analysis
- [ ] Partnership pipeline review
- [ ] Roadmap adjustment based on learnings
- [ ] Update investor narrative and materials
- [ ] Team/contractor performance evaluation

## Crisis Mode Protocol

When things break (and they will), switch to **Crisis Mode**:

1. **Assess**: Is this revenue-impacting? User-facing? Data-integrity?
2. **Communicate**: Update affected users within 30 minutes
3. **Fix**: Drop everything non-critical until resolved
4. **Post-mortem**: Document what happened and prevention plan
5. **Resume**: Return to routine next day — don't let crises derail momentum

## The Compound Effect

> "You don't build a proptech empire in a day. You build it with 365 days of disciplined execution."

Daily consistency compounds:
- **1 feature/day** = 30 improvements/month = unrecognizable platform in 6 months
- **3 developer contacts/day** = 90/month = dominant supply in key markets
- **1 content piece/day** = organic traffic flywheel within 3 months
- **1 investor touch/day** = warm pipeline ready when fundraising

The routine is the strategy. Execute it relentlessly.
