# ASTRAVILLA Investor Demo — UI Storytelling Script
**Date:** 2026-03-16 | **Duration:** 8–10 minutes live demo
**Format:** Sequential screen flow with talking points per scene
**Rule:** Every click must reveal intelligence. No dead screens.

---

## Pre-Demo Setup

- Browser: Chrome, dark mode enabled (premium perception is strongest in dark mode)
- Viewport: 1440×900 (investor laptop standard)
- Data: Ensure Bali properties have images, AI scores, and price history populated
- Tabs pre-loaded: Homepage, a Canggu villa detail page, Market Intelligence, Investor Dashboard
- Close all other browser tabs — full attention on ASTRAVILLA

---

## Demo Anti-Patterns (Things to Avoid)

| Don't | Why | Instead |
|-------|-----|---------|
| Show admin dashboard to investors | Looks like internal tooling, not product | Show consumer + investor-facing intelligence |
| Click through 10+ pages | Feels like a feature tour, not a story | 5-6 screens maximum, each with a narrative purpose |
| Show empty states or loading errors | Destroys confidence | Pre-load all demo pages, use populated data |
| Read text on screen aloud | Insulting to audience intelligence | Point at visuals, speak the insight behind them |
| Demo on mobile phone | Too small for group viewing | Demo on laptop at 1440px, mention mobile separately |
| Show unfinished features | "Coming soon" signals immaturity | Only show what works end-to-end today |
| Spend time on basic CRUD | Listing creation is commodity | Focus on intelligence outputs, not data entry |
